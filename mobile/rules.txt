## üß© COMPONENT ARCHITECTURE

### Design Principles
- **Consistency**: Use predefined variants and sizes
- **Accessibility**: Proper focus states and ARIA labels
- **Performance**: Minimal re-renders with proper state management
- **Maintainability**: Reusable components with clear APIs

### Component Structure
- **Props Interface**: Clear TypeScript interfaces
- **Default Values**: Sensible defaults for all props
- **Variants**: Use cva (class-variance-authority) for variants
- **Composition**: Prefer composition over inheritance

### State Management
- **Local State**: useState for component-specific state
- **Global State**: Zustand for app-wide state
- **Server State**: React Query for API data
- **Form State**: React Hook Form for form management

## üé® ICON SYSTEM

### Icon Library
- **Primary**: Lucide React (lucide-react)
- **Size Classes**: w-4 h-4, w-5 h-5, w-6 h-6
- **Color**: Inherit from parent or use semantic colors
- **Alignment**: flex items-center gap-2 for icon + text

### Icon Usage
- **Action Icons**: Use semantic colors (success, warning, error)
- **Decorative Icons**: Use muted colors
- **Interactive Icons**: Include hover states
- **Accessibility**: Always include aria-label for screen readers

## üîß UTILITY CLASSES

- **twMerge**: For merging Tailwind classes without conflicts

### Layout Utilities
- **Container**: max-w-7xl mx-auto px-6
- **Section**: space-y-6
- **Card**: bg-card border rounded-xl shadow-sm
- **Button**: Use buttonVariants for consistent styling

## üìã IMPLEMENTATION GUIDELINES

### Do's
- ‚úÖ Use predefined color variables from CSS
- ‚úÖ Follow the spacing system (4px increments)
- ‚úÖ Use component variants instead of custom classes
- ‚úÖ Implement proper hover and focus states
- ‚úÖ Use semantic color names for different states
- ‚úÖ Follow responsive design patterns

### Don'ts
- ‚ùå Don't use hardcoded colors
- ‚ùå Don't create custom spacing values
- ‚ùå Don't override component variants unnecessarily
- ‚ùå Don't skip hover and focus states
- ‚ùå Don't use non-semantic colors for states
- ‚ùå Don't ignore responsive design

### File Organization
- **Components**: /components/ui/ for base components
- **Features**: /features/ for feature-specific components
- **Utils**: /lib/utils.ts for utility functions
- **Types**: /types/ for TypeScript interfaces
- **Hooks**: /hooks/ for custom React hooks

This design system ensures consistency, accessibility, and maintainability across the entire Real State frontend application.
